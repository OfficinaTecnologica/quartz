Git è come una macchina del tempo per il codice: ricorda ogni cambiamento, così se qualcosa esplode (ops) puoi tornare indietro e vedere chi ha fatto cosa.

Il posto dove vive un progetto si chiama repository (repo): immagina uno zaino/cassaforte digitale che contiene i file e tutta la loro storia.

Con una repo puoi:
- creare “salvataggi” (commit) e recuperare versioni vecchie;
- confrontare le differenze tra versioni per stanare il bug;
- lavorare in squadra senza pestarsi i piedi: ognuno su un proprio ramo (branch) e poi si uniscono.

Le repo possono stare sul tuo computer o su servizi online che usano Git, tipo GitHub, così puoi collaborare con gli altri e avere un backup al sicuro.
